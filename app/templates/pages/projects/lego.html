Visual Programming
Create animations, solve problems, generate Python/C++ code with drag-and-drop blocks.
PROBLEM

Many beginner programmers struggle with syntax, spending more time worrying about writing code correctly rather than focusing on problem-solving and logical thinking. We realized that this focus on syntax often hinders the learning process, preventing them from fully understanding how to solve problems effectively. This project was created to shift that focus entirely to logic. By dragging and dropping blocks, users can apply their logic visually without writing a single line of code. Additionally, creating visual outputs or drawings is just as easy, allowing for a seamless, syntax-free learning experience.

The Blocks, How does the app generate code from the blocks, and how are drawings created?

1

First, select blocks from the block list and drag them into the code area to build your logic. Each block represents a fundamental coding concept, like loops, conditions, or functions, allowing you to visually structure your program.

2

Block Stage: Each block is then processed in this stage, where the text is extracted and its unique ID is mapped to predefined identifiers. For example, special variables are mapped to types like 'allFatherL' for lists and 'allFather' for primitive types. This mapping ensures that the app understands the intended functionality of each block within the generated code.

3

Lexical Stage: the lexical analysis stage involves scanning the input code to convert it into a sequence of tokens, which are the basic building blocks of the code. The output from the Block Stage is then tokenized into an object, assigning a unique number to each token along with its corresponding value.

4

Parser Stage:In this stage, the code is analyzed to determine its grammatical structure. During parsing, a symbol table is created, which maintains a record of all the objects that the list holds, as well as the environment in which the list is defined. This allows the app to keep track of the scope, ensuring that each variable or function can be accessed appropriately within its defined context. The symbol table serves as a crucial data structure in the parser, storing information about identifiers, such as variables and functions, along with their attributes like data type, scope level, and memory location. This enables efficient management of variable states and helps prevent naming conflicts.

AST: The parser also generates an Abstract Syntax Tree (AST) that represents the hierarchical structure of the code. From this AST, the app checks the grammars for both C++ and Python. Notice that there's no need to define variable types or the function return type in C++; these are auto-generated after creating and traversing the AST. During this traversal, the app verifies various aspects, such as:

  • Whether arithmetic expressions are valid
  • Whether variables are defined and accessible within their scope
  • Whether variable names are permissible
    • Additionally, the parser can prevent minor errors by assigning default values to variables that the user did not define. This ensures that the code runs smoothly without unexpected behavior. Throughout this process, the app provides live feedback to the user, highlighting any errors in the code to facilitate a smoother programming experience.
    5

    Code Generation: After validating the code, the Abstract Syntax Tree (AST) is converted into C++ and Python code by mapping the predefined variables and structures into suitable constructs for each language. This mapping ensures that the logical flow and functionality represented in the AST are accurately translated into syntactically correct code that adheres to the rules and conventions of both C++ and Python. The result is a seamless generation of executable code that reflects the user's intended logic as represented by the blocks.

    The Blocks, Drawing

    After generating the executable code in Python, a WebSocket communication is established between the backend and the frontend. Each drawing command generated from the blocks is sent to JavaScript for processing. This allows the app to control the drawing interface, moving the turtle to specified coordinates and executing drawing instructions such as creating lines, shapes, or other visual elements. As each block is processed, the corresponding visual output is rendered in real time, providing users with immediate feedback and allowing them to see their logic and creativity come to life through interactive drawings.
