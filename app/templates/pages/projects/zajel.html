Zajel Mobile
Cross Mobile Platform App . 2024
Zajel Mobile is a mobile application developed for university students, providing a faster and easier way to access their academic data. The app enhances the user interface and experience, supports both offline and online modes, and sends notifications to students about new updates and upcoming lectures.

PROBLEM

The previous version of the Zajel Mobile app faced several challenges that led to negative reviews from users. These issues included a lack of full synchronization with the Zajel Web platform, frequent unexpected logouts, and difficulties handling internet connectivity. Additionally, users experienced slow response times, problems with fetching data, and inconsistent delivery of notifications, which affected the overall user experience.

Given the numerous issues with the previous version, I decided to completely recreate the Zajel Mobile app instead of just fixing the existing bugs. One of my main priorities was to significantly improve the user interface and experience. I focused on redesigning key features, such as the student curriculum page, which required a more efficient way to handle and display large amounts of data. I implemented a filter search to make it easier for users to find specific information and reorganized the layout for better readability and usability.

    RESEARCH CASE: Enhancing Student Engagement and Experience

    I conducted research to identify pain points in the student experience, particularly related to class schedules and borrowed library books. I found that many students were missing their lectures because they had to repeatedly log in to the university's web portal or the outdated mobile app just to check the time and room details for their classes. This process was not user-friendly and led to students being late or even missing lectures entirely. Additionally, I discovered that students often failed to return borrowed library books on time, resulting in overdue fees. This issue stemmed from a lack of reminders about the approaching return deadlines.

    Frontend: Expo-React Native. Why Expo?

  • Deep Linking: Directs users to specific in-app locations via custom URL schemes, improving navigation and reducing friction.
  • Fast Refresh: Instantly reflects code changes across Android, iOS, and web without a full reload.
  • Offline Support: Caches visited pages, allowing the app to work offline.
  • Over-the-Air (OTA) Updates: Pushes updates directly to users without requiring an app store release, useful for quick bug fixes and UI changes.
    • 2

    Backend: Javalin

  • Scalable Architecture: Supports WebSockets, async processing, and efficient request handling.
  • Lightweight and Fast Setup: Easy configuration and flexible routing, ideal for microservices.
    • 3

    Why We Need A Backend Service?

  • User Authentication: Manages permissions to ensure secure data access.
  • Push Notifications: Sends updates and reminders to students.
  • Account Security: Alerts students if their account is accessed from a new device.
    • Results
    The implementation led to a noticeable increase in user satisfaction, as students now enjoy a smoother and more intuitive way to navigate and access information. Network issues were effectively resolved, with offline support and optimized data fetching ensuring reliable app performance even with unstable or no internet connectivity. The integration of deep linking and enhanced navigation provides students with an efficient way to quickly find the information they need. Additionally, push notifications keep students informed by ensuring they never miss important updates, reminders, or announcements.
